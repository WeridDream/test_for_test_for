# CHG-001 表单平台路由策略采用 A（protect-by-default）（SoT）
- Doc ID: CHG-001
- Status: FINAL
- Owner: IU
- Last Updated: 2026-02-25
- Revision: r20260225_01
- Supersedes: N/A
- Depends On: FL-001, DEP-ENV-001, SEC-002

## 1. Why（必须）
- 选定表单平台门禁路由策略 A：对 form_platform upstream 默认 fail-closed（必须 JWT），仅放行极少数入口（/_auth/* 等）。
- 目的：避免依赖“枚举敏感 URL 列表”，降低第三方平台升级/隐藏接口导致的漏拦截风险。

## 2. Impact（必须）
- FL-001：流程描述需要体现 “先 gate 种 Cookie → 后续所有表单请求带 Cookie → Envoy 本地验签”。
- DEP-ENV-001（envoy.yaml）：路由与 jwt_authn rules 需要按 A 调整。
- SEC-002：补充“签发/验证分层”不变量（Rust 不参与每请求鉴权）。

## 3. Diff（必须）
- 新增/更新条款见：FL-001 7.1、DEP-ENV-001 3.x、SEC-002 2.x

## 4. Rollback（必须）
- 回滚仅需修改 Envoy route：从“默认 require JWT”回滚为“仅保护 /s,/q/ 等关键路径”。
- 不涉及 DB/契约变更，无数据迁移风险。
